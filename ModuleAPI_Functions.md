# ModuleAPI 动态库函数说明

本文档列出了 `libModuleAPI.so` 中导出的主要函数及其功能说明。

## 核心 C API 函数 (Core C API)
这些是标准的 C 风格导出函数，也是 Python (`ctypes`) 调用该库的主要接口。

### 1. 生命周期管理 (Lifecycle Management)
*   **`InitReader`**
    *   **功能**: 初始化读写器实例并建立连接。
    *   **说明**: 通常需要传入连接字符串（如串口号 `/dev/ttyUSB0` 或 IP 地址）。
*   **`InitReader_Notype`**
    *   **功能**: 自动检测类型的初始化。
    *   **说明**: 相比 `InitReader`，可能不需要指定具体的读写器型号。
*   **`CloseReader`**
    *   **功能**: 关闭连接并释放资源。
    *   **说明**: 程序退出前必须调用，否则可能导致下次连接失败（串口被占用）。
*   **`RebootReader`**
    *   **功能**: 重启读写器设备。
    *   **说明**: 软重启设备，用于从异常状态恢复。
*   **`GetDeviceVersion`**
    *   **功能**: 获取设备版本信息。
    *   **说明**: 返回固件版本、硬件版本号。

### 2. 标签操作/盘点 (Tag Operations / Inventory)
*   **`TagInventory`**
    *   **功能**: 开始标签盘点（读取）。
    *   **说明**: 标准的盘点模式，读写器开始搜索周围的 RFID 标签。
*   **`TagInventory_Raw`**
    *   **功能**: 原始盘点模式。
    *   **说明**: 演示脚本中使用的模式，可能提供更底层的控制或返回更详细的原始数据。
*   **`TagInventory_Single`**
    *   **功能**: 单次读取。
    *   **说明**: 尝试读取一个标签后立即停止，适用于触发式读取。
*   **`TagInventory_Count`**
    *   **功能**: 获取已读取的标签数量。
*   **`GetNextTag`**
    *   **功能**: 获取下一个标签数据。
    *   **说明**: 从内部缓冲区中取出一个标签信息（EPC、RSSI、天线号等）。通常在循环中调用。
*   **`AsyncGetNextTag`**
    *   **功能**: 异步获取下一个标签。
    *   **说明**: 非阻塞方式获取标签数据。

### 3. 参数配置 (Configuration)
*   **`ParamSet`**
    *   **功能**: 通用参数设置函数。
    *   **说明**: 通过 Key-Value 的方式设置各种参数（如功率、频率、协议等）。这是最常用的配置入口。

## 高级功能与内部方法 (Advanced Capabilities)
以下功能主要对应 C++ 类 `Sl_Reader` 的方法。虽然通过 C API (`ParamSet`) 可能间接调用，但了解这些名称有助于理解硬件支持的功能。

### 射频与功率 (RF & Power)
*   **`Set_ReadPower` / `Set_WritePower`**
    *   **功能**: 设置读取/写入功率。
    *   **说明**: 功率越大，读取距离越远，但干扰也可能越大。单位通常是 dBm。
*   **`Set_AntsPower`**
    *   **功能**: 独立设置每个天线的功率。
    *   **说明**: 针对多天线设备（如 4口/8口读写器），可以为不同天线设置不同功率。
*   **`Get_AntPortsVSWR`**
    *   **功能**: 获取天线端口驻波比 (VSWR)。
    *   **说明**: 用于检测天线连接质量。驻波比过高说明天线未接好或损坏。
*   **`Set_Region` / `Get_Region`**
    *   **功能**: 设置/获取频率区域。
    *   **说明**: 设置符合当地法规的频率范围（如中国 920-925MHz, 美国 FCC, 欧洲 ETSI）。
*   **`Set_FrequencyHopTable`**
    *   **功能**: 设置跳频表。
    *   **说明**: 自定义具体的频率跳变列表。

### Gen2 协议参数 (Gen2 Protocol)
*   **`Set_Gen2Session`**
    *   **功能**: 设置会话层 (Session S0/S1/S2/S3)。
    *   **说明**: 决定标签被读取后的静默行为。
        *   S0: 标签读取后立即复位（适合少量标签快速连续读取）。
        *   S1-S3: 标签读取后保持静默一段时间（适合多标签盘点，防止重复读取同一个标签占用时间）。
*   **`Set_Gen2Qval`**
    *   **功能**: 设置 Q 值 (防碰撞算法参数)。
    *   **说明**:
        *   Q 值越小：适合标签数量少的情况，响应快。
        *   Q 值越大：适合标签数量多的情况，减少碰撞概率。
*   **`Set_Gen2Target`**
    *   **功能**: 设置读取目标 (Target A/B)。
    *   **说明**: 配合 Session 使用，翻转标签的 A/B 状态，实现盘点循环。

### GPIO 控制 (General Purpose I/O)
*   **`Set_GPO`**
    *   **功能**: 设置通用输出端口 (Output)。
    *   **说明**: 可以控制连接在读写器上的外部设备，如报警灯、继电器、电锁等。
*   **`Get_GPI`**
    *   **功能**: 获取通用输入端口 (Input) 状态。
    *   **说明**: 读取外部传感器（如光电开关、红外触发）的状态。

### 标签特殊操作
*   **`Write_TagData_Imp`**
    *   **功能**: 向标签写入数据。
    *   **说明**: 修改标签的 User 区或 EPC 区数据。
*   **`Lock_Tag`**
    *   **功能**: 锁定标签。
    *   **说明**: 锁定特定内存区（如 Access Password, EPC），防止被修改。
*   **`Kill_Tag`**
    *   **功能**: 销毁标签。
    *   **说明**: 永久禁用标签（Kill 命令），使其不再响应任何指令。
*   **`Read_TagTemperature`**
    *   **功能**: 读取标签温度。
    *   **说明**: 仅适用于带温度传感器的特殊标签。

### 过滤与筛选
*   **`SetSubTagSelector` / `Set_MultiTagSelector`**
    *   **功能**: 设置标签过滤器。
    *   **说明**: 只读取符合特定规则（如 EPC 前缀匹配）的标签，忽略其他标签。

### 系统设置
*   **`Set_Watchdog`**
    *   **功能**: 设置看门狗。
    *   **说明**: 防止读写器死机，超时自动重启。
*   **`Set_IpInfo`**
    *   **功能**: 设置网络参数。
    *   **说明**: 修改读写器的 IP 地址、掩码、网关（针对网口设备）。
*   **`Get_Rfid_Temperature`**
    *   **功能**: 获取读写器内部温度。
    *   **说明**: 监控设备运行温度，防止过热。
